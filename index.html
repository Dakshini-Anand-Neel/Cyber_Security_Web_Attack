Beginner's Guide: What are we detecting?

Normal Traffic

Everyday web requests. Users logging in normally, searching for products, or filling out standard contact forms. These payloads usually contain standard text, names, and regular characters.

XSS (Cross-Site Scripting)

Attackers try to sneak malicious JavaScript code into a website. If successful, this code runs in *other users'* browsers, potentially stealing their login cookies or hijacking their sessions. Look for tags like <script>.

SQLi (SQL Injection)

Attackers try to trick the website's database. By entering database commands (like SELECT or OR 1=1) into input fields, they attempt to bypass logins, steal private data, or even delete the entire database.

Why use Machine Learning?

Traditional security relies on "rule lists" (e.g., Block anything with <script>). Hackers easily bypass these by slightly changing their text (e.g., <sCrIpT>). Our Machine Learning pipeline acts like an AI detective, learning the underlying *patterns* and *structure* of attacks rather than just memorizing rules, making it much harder to trick.

The ML Goal: Notice how the colors clump together? Our Machine Learning model (Step 6) learns to draw mathematical boundaries between these distinct clusters to classify new, unseen traffic.
